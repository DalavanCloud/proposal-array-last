Last item from Arrays

Common Footguns

            
// Just calling `myArray.length` always gets `undefined`
myArray[myArray.length] 

// Overcooking indexes to be too far:
myIndex = myArray.length  - 1
myArray[myIndex - 1] // returns last but one, not last

Solution 1: Array#-1

            
myArray[-1] === myArray[myArray.length - 1]

Solution 1: Array#-1 Pros

  • Concise syntax
  • Can be a getter/setter, still intuitive
  • Behaves like normal property access
  • Precedent from other languages, see: Python
  • Could extend being a right-hand lookup op, i.e. -2, -3 etc.

Solution 1: Array#-1 Cons

  • Computed property access is still a footgun
  • Can accidentally hit this property with off-by-one-errors

Solution 2: Array#:-1

            
myArray[:-1] === myArray[myArray.length - 1]

// Maybe?
myArray[:-2] === myArray[myArray.length -2]
myArray[:-6] === myArray[myArray.length -6]
myArray[1:-1] === myArray.slice(1, -1)
myArray[2:3] === myArray.slice(2, 3)
myArray[-1:1] === myArray.slice(1, -1).reverse()

Solution 2: Array#:-1 Pros

  • Segues into range access; slicing of Arrays: [2:-2], [2:-0] etc.
  • Could also segue to immutable reverse: [-0:0], [-3, 3] etc.
  • Similar precedent from other languages, see: Ruby (a[2, -2], a[-3, 3])

Solution 2: Array#:-1 Cons

  • New syntax
  • Hard to spec
  • Impossible to polyfill (could be ponyfilled with Proxies with string based indexing)
  • Computed property access is still a footgun

Solution 3: Array#last()

            
myArray.last() === myArray[myArray.length - 1]

Solution 3: Array#last() Pros

  • Very simple to spec (could go to stage 2 today)
  • Hard to mistakenly do, as opposed to negative indexes
  • Behaves like normal property access
  • Precedent from other languages, see: Ruby(ish), LINQ, PHP (end())
  • Marries well with lastIndexOf

Solution 3: Array#last() Cons

  • Slippery slope to add more methods, first(), second(), fourty_two()
  • Slippery slope for adding arguments (e.g. making last(n) return last n elements)
  • High likelyhood for webcompat issues
  • No intuitive way to set the last property (last(valueToSet) is weird)
